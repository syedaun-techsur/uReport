<objective>
Ensure the dev server (`npm run dev`, launched via `.pivota/start-dev.sh`) applies migrations and seeds the database before Next.js starts — the same guarantee that `npm start` (via `scripts/migrate-and-start.js`) already provides.

Purpose: UAT Gap 1 root cause includes "DB was not yet migrated+seeded when user first tested." The production start script (`scripts/migrate-and-start.js`) runs `prisma migrate deploy` + auto-seed. But `start-dev.sh` goes directly to `npm run dev` with no DB preparation. On a fresh sandbox (or after a schema change), the User table may not exist or may be empty, causing every login to fail before auth logic is even reached. Adding an idempotent pre-launch block to `start-dev.sh` closes this gap for dev mode without changing the production path.

Output:
- `.pivota/start-dev.sh` — new migrate+seed block inserted between the D-12 install section and the D-14 retry loop (above `# === END PIVOTA PREAMBLE ===`)
</objective>

Insert the following block IMMEDIATELY BEFORE that line (with one blank line of separation):

```bash
# === Pre-launch: migrate + seed (dev-mode DB prep) ===
# mirrors scripts/migrate-and-start.js for 'npm run dev' path.
# Runs prisma migrate deploy (idempotent) then seeds if User table is empty.
# Skipped entirely when DATABASE_URL is not set (no sidecar / compose DB).
if [ -n "${DATABASE_URL:-}" ]; then
  echo "[pivota-dev-db] Running prisma migrate deploy..."
  npx prisma migrate deploy || echo "[pivota-dev-db] migrate non-zero (may be OK if schema already current)"

  echo "[pivota-dev-db] Checking if seed is needed..."
  USER_COUNT=$(node -e "
    const { Client } = require('pg');
    const c = new Client({ connectionString: process.env.DATABASE_URL });
    c.connect()
      .then(() => c.query('SELECT COUNT(*)::int AS cnt FROM \"User\"'))
      .then(r => { console.log(r.rows[0].cnt); return c.end(); })
      .catch(() => { console.log('0'); });
  " 2>/dev/null || echo "0")
  if [ "${USER_COUNT:-0}" -eq 0 ]; then
    echo "[pivota-dev-db] Empty DB — running seed..."
    npx tsx prisma/seed.ts || echo "[pivota-dev-db] seed non-zero (check seed output above)"
  else
    echo "[pivota-dev-db] Seed skipped — ${USER_COUNT} user(s) already exist."
  fi
fi
```

Notes:
- `[ -n "${DATABASE_URL:-}" ]` — the `:-` default prevents `set -u` from aborting when DATABASE_URL is unset.
- `migrate deploy` exit is non-fatal (`|| echo ...`) because "no pending migrations" still exits 0, and any real failure will surface in the dev server logs immediately.
- The user count query uses Node + pg (already a project dependency) rather than `psql` which may not be in PATH inside the sandbox.
- The block is placed ABOVE `# === END PIVOTA PREAMBLE ===` inside the preamble region — this is intentional. The preamble comment says "Edits ABOVE the END PIVOTA PREAMBLE marker are reset on regen," but since this is a deliberate project decision (not auto-generated), document in the comment that this section should be preserved.
