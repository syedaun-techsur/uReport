**app/api/staff/tickets/[id]/route.ts** — Add PATCH handler (GET exists from 05-03):

`params`: `Promise<{ id: string }>` — await per Next.js 15 pattern.

```
const UpdateTicketSchema = z.object({
  status: z.enum(['open', 'in_progress', 'closed', 'archived']).optional(),
  substatus_id: z.string().cuid().nullable().optional(),
  assignee_id: z.string().cuid().nullable().optional(),
  note: z.string().max(4000).optional(),
  department_id: z.string().cuid().nullable().optional(),
});
```

PATCH flow:
1. `await requireSession('staff')` — get session with `user.id` and `user.role`.
2. Parse body with `UpdateTicketSchema`. On error: 422.
3. `const { id } = await params`.
4. Fetch current ticket: `prisma.ticket.findUnique({ where: { id }, select: { status: true, assignee_id: true, department_id: true } })`. If null: 404.
5. **Status transition guard**: if `status === 'open'` and current ticket `status === 'closed'` and `session.user.role !== 'admin'`: return `apiError('TRANSITION_FORBIDDEN', 'Only admins can re-open closed tickets', 403)`.
6. **Substatus validation**: if `substatus_id` provided (non-null) and `status` provided, verify `prisma.substatus.findFirst({ where: { id: substatus_id, status: newStatus } })` — if null return `apiError('SUBSTATUS_MISMATCH', 'Substatus does not belong to selected status', 422)`.
7. Use `prisma.$transaction([...])` to atomically:
   - `prisma.ticket.update({ where: { id }, data: { status?, substatus_id?, assignee_id?, department_id?, updated_at: new Date() } })`
   - For status change: `prisma.ticketHistory.create({ data: { ticket_id: id, actor_id: session.user.id, action: 'STATUS_CHANGE', from_value: oldStatus, to_value: newStatus } })`
   - For assignee change: `prisma.ticketHistory.create({ data: { ticket_id: id, actor_id: session.user.id, action: 'ASSIGNMENT', from_value: oldAssigneeId, to_value: newAssigneeId } })`
   - For department change: `prisma.ticketHistory.create({ data: { ticket_id: id, actor_id: session.user.id, action: 'ASSIGNMENT', from_value: oldDeptId, to_value: newDeptId, note: 'Department updated' } })`
   - If `note` provided: `prisma.response.create({ data: { ticket_id: id, author_id: session.user.id, body: note, is_public: false } })`
8. Re-fetch full ticket with same includes as GET and return `ok(ticketDetailShape)`.

**app/api/staff/tickets/[id]/responses/route.ts**:

`params`: `Promise<{ id: string }>` — await.

```typescript
const CreateResponseSchema = z.object({
  body: z.string().min(1).max(10000).refine(s => s.trim().length > 0, { message: 'Response body cannot be empty' }),
  is_public: z.boolean(),
  template_id: z.string().cuid().optional(),
});
```

POST flow:
1. `await requireSession('staff')`.
2. `const { id: ticketId } = await params`.
3. Verify ticket exists: `prisma.ticket.findUnique({ where: { id: ticketId }, select: { id: true } })`. If null: 404.
4. Parse body with `CreateResponseSchema`. On empty body: `apiError('EMPTY_RESPONSE', 'Response body cannot be empty', 422)`.
5. `prisma.$transaction([
     prisma.response.create({ data: { ticket_id: ticketId, author_id: session.user.id, body: parsed.body, is_public: parsed.is_public, template_id: parsed.template_id ?? null } }),
     prisma.ticketHistory.create({ data: { ticket_id: ticketId, actor_id: session.user.id, action: 'RESPONSE', to_value: parsed.is_public ? 'public' : 'internal', note: parsed.body.slice(0, 200) } }),
     prisma.ticket.update({ where: { id: ticketId }, data: { updated_at: new Date() } }),
   ])`.
6. Return `ok({ response_id: response.id, body: response.body, is_public: response.is_public, created_at: response.created_at.toISOString() }, 201)`.

**app/api/staff/tickets/[id]/media/route.ts**:

`params`: `Promise<{ id: string }>` — await.

POST flow:
1. `await requireSession('staff')`.
2. `const { id: ticketId } = await params`.
3. Verify ticket: `prisma.ticket.findUnique({ where: { id: ticketId }, select: { id: true } })`. If null: 404.
4. Parse multipart: `const formData = await req.formData(); const files = formData.getAll('files') as File[]`.
5. Validate: if `files.length === 0`: 422; if `files.length > 5`: `apiError('MEDIA_TOO_MANY', 'Maximum 5 files per upload', 422)`.
6. For each file:
   - Check size: if `file.size > 10 * 1024 * 1024`: `apiError('MEDIA_TOO_LARGE', 'File exceeds 10MB limit', 422)`.
   - Check MIME: if not `image/*` or `application/pdf`: `apiError('MEDIA_TYPE_INVALID', 'File type not accepted', 422)`.
7. In a Prisma transaction:
   - For each file: call `storeMedia(prisma, ticketId, file)` from `@/lib/media.ts` (Phase 3 pattern).
   - `prisma.ticketHistory.create({ data: { ticket_id: ticketId, actor_id: session.user.id, action: 'MEDIA_ADDED', to_value: files.map(f => f.name).join(', ') } })`.
   - `prisma.ticket.update({ where: { id: ticketId }, data: { updated_at: new Date() } })`.
   Note: `storeMedia` itself creates the Media record. Wrap the entire loop + history create in `prisma.$transaction`.
8. Return `ok({ media: createdMedia.map(m => ({ media_id: m.id, filename: m.filename, mime_type: m.mime_type, created_at: m.created_at.toISOString() })) }, 201)`.
