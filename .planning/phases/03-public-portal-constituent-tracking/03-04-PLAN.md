The current migration gate in .pivota/start-dev.sh (in the PREAMBLE section, before the `# === END PIVOTA PREAMBLE ===` marker) reads:

```bash
if [ -n "${PIVOTA_DB_MODE:-}" ] || [ -n "${DATABASE_URL:-}" ]; then
  if node -e "process.exit(require('./package.json').scripts?.migrate?0:1)" 2>/dev/null; then
    echo "[pivota] platform DB detected — running 'npm run migrate' (best-effort) before dev server"
    npm run migrate || echo "[pivota] migrate non-zero (continuing; dev script may re-run it)"
  fi
fi
```

This checks for a `scripts.migrate` key, but the project declares `scripts.db:migrate`. The inner `if` never fires.

**Fix:** Replace the inner script-name check entirely with an unconditional `npx prisma migrate deploy && npx tsx prisma/seed.ts || true` when DATABASE_URL is present. This is the same pattern as the 02-08 gap fix that placed a block *below* the END PIVOTA PREAMBLE marker.

**Decision context (from STATE.md):** The Phase 02-08 fix added migrate+seed *below* the END PIVOTA PREAMBLE marker so it survives start-dev.sh regeneration. However, the broken gate sits *above* the marker in the preamble. To avoid conflicting logic (preamble gate still runs first, does nothing, then the below-marker block runs), replace the broken inner check in-place — change the `node -e` condition to check for `db:migrate` instead, AND update the `npm run migrate` call to `npm run db:migrate`.

Exact replacement in the preamble block (do NOT touch the preserved region below END PIVOTA PREAMBLE):

OLD:
```bash
  if node -e "process.exit(require('./package.json').scripts?.migrate?0:1)" 2>/dev/null; then
    echo "[pivota] platform DB detected — running 'npm run migrate' (best-effort) before dev server"
    npm run migrate || echo "[pivota] migrate non-zero (continuing; dev script may re-run it)"
  fi
```

NEW:
```bash
  if node -e "process.exit(require('./package.json').scripts?.['db:migrate']?0:1)" 2>/dev/null; then
    echo "[pivota] platform DB detected — running 'npm run db:migrate' (best-effort) before dev server"
    npm run db:migrate || echo "[pivota] db:migrate non-zero (continuing; dev script may re-run it)"
    echo "[pivota] running seed (best-effort, non-fatal)"
    npm run db:seed || echo "[pivota] db:seed non-zero (continuing; DB may already be seeded)"
  fi
```

Using `scripts?.['db:migrate']` with bracket notation is required because `db:migrate` contains a colon which is not valid as a property accessor via dot notation in JavaScript.

Also add a seed call **after** the migrate call so categories are populated from the CategoryGroups/Categories seed data.

Note: The preserved region *below* the END PIVOTA PREAMBLE marker was added by 02-08 and already runs `npx prisma migrate deploy && npx tsx prisma/seed.ts`. After this fix, BOTH blocks will fire on the next boot — that is safe (idempotent migrations) and desirable as belt-and-suspenders. Do NOT remove the below-marker block.
