**1. Create `schemas/admin.ts`** with all admin Zod schemas:

```typescript
import { z } from 'zod';

export const CreateCategorySchema = z.object({
  service_code: z.string().min(1).max(50).regex(/^[a-zA-Z0-9_]+$/, 'Only alphanumeric and underscores'),
  name: z.string().min(1).max(200),
  description: z.string().max(2000).optional(),
  icon: z.string().max(100).optional(),
  color: z.string().regex(/^#[0-9A-Fa-f]{6}$/, 'Must be #RRGGBB').optional().nullable(),
  anon_allowed: z.boolean().default(true),
  active: z.boolean().default(true),
  group_id: z.string().cuid().optional().nullable(),
  department_id: z.string().cuid().optional().nullable(),
  sla_hours: z.number().int().positive().optional().nullable(),
});

export const UpdateCategorySchema = CreateCategorySchema.partial();

export const CreateDepartmentSchema = z.object({
  name: z.string().min(1).max(200),
  default_assignee_id: z.string().cuid().optional().nullable(),
  active: z.boolean().default(true),
});

export const UpdateDepartmentSchema = CreateDepartmentSchema.partial();

export const CreateSubstatusSchema = z.object({
  label: z.string().min(1).max(100),
  internal_label: z.string().max(100).optional().nullable(),
  status: z.enum(['open', 'in_progress', 'closed', 'archived']),
  sort_order: z.number().int().min(0).default(0),
  active: z.boolean().default(true),
});

export const UpdateSubstatusSchema = CreateSubstatusSchema.partial();

export const ReorderSubstatusSchema = z.object({
  items: z.array(z.object({
    id: z.string().cuid(),
    sort_order: z.number().int().min(0),
  })).min(1),
});

export const CreateTemplateSchema = z.object({
  name: z.string().min(1).max(200),
  body: z.string().min(1).max(10000),
  category_id: z.string().cuid().optional().nullable(),
  department_id: z.string().cuid().optional().nullable(),
  active: z.boolean().default(true),
});

export const UpdateTemplateSchema = CreateTemplateSchema.partial();
```

**2. Create route handlers** — ALL must:
- Import `{ requireSession }` from `'@/lib/api-response'`
- Import `{ prisma }` from `'@/lib/prisma'` (named import, NOT default)
- Call `const session = await requireSession('admin')` first; if `session instanceof NextResponse` return it immediately
- Use `params: Promise<{ id: string }>` and `const { id } = await params` for dynamic segments (Next.js 15 pattern)
- Create `AdminAuditLog` entry in the SAME Prisma transaction as the mutation (not a separate call after)
- Return errors using `apiError(code, message, status)` from `@/lib/api-response`
- Return success using `ok(data, status)` from `@/lib/api-response`

**`app/api/admin/categories/route.ts`** (GET + POST):
- GET: return `{ data: Category[], meta: { total, page, page_size, total_pages } }`. Include `group_id`, `group` (name only), `department_id`, `department` (name only) via Prisma `include`. Support query param `?active=true|false|all` (default returns all). Support `?page` and `?page_size` (default 25, max 100).
- POST: parse `CreateCategorySchema`. Check `service_code` uniqueness across ALL categories (not just active). Check `department_id` references active dept. Check `group_id` references existing CategoryGroup. Create category + AdminAuditLog in one transaction: `action = 'CATEGORY_CREATED'`, `resource_type = 'Category'`, `resource_id = category.id`, `metadata = { name, service_code }`.

**`app/api/admin/categories/[id]/route.ts`** (PATCH):
- Parse `UpdateCategorySchema`. Check `service_code` uniqueness excluding current record.
- On `active = false`: count open tickets referencing this category and include `open_ticket_count` in response (warning only, allow deactivation).
- Transaction: update category + AdminAuditLog with `action = 'CATEGORY_UPDATED'` or `'CATEGORY_DEACTIVATED'` based on whether `active` changed to false.
- `metadata` must NOT include PII; include field names and new values that are non-PII.

**`app/api/admin/departments/route.ts`** (GET + POST):
- GET: return all departments with `users` count (staff members linked). Support `?active=true|false|all`.
- POST: parse `CreateDepartmentSchema`. Check dept `name` uniqueness among active depts. Check `default_assignee_id` is active user. Create + AdminAuditLog (`action = 'DEPARTMENT_CREATED'`).

**`app/api/admin/departments/[id]/route.ts`** (PATCH):
- On `active = false`: count active categories linked to dept, include `active_category_count` in response.
- AdminAuditLog: `action = 'DEPARTMENT_UPDATED'` or `'DEPARTMENT_DEACTIVATED'`.

**`app/api/admin/substatuses/route.ts`** (GET + POST):
- GET: return all substatuses ordered by `status`, then `sort_order`. Support `?status=open|in_progress|closed|archived` filter.
- POST: check `label` uniqueness within same `status`. AdminAuditLog: `action = 'SUBSTATUS_CREATED'`.

**`app/api/admin/substatuses/[id]/route.ts`** (PATCH):
- AdminAuditLog: `action = 'SUBSTATUS_UPDATED'`.

**`app/api/admin/substatuses/reorder/route.ts`** (PATCH):
- Parse `ReorderSubstatusSchema`. Update each substatus `sort_order` in a single transaction using `prisma.$transaction(items.map(...))`. AdminAuditLog single entry: `action = 'SUBSTATUS_REORDERED'`, `resource_type = 'Substatus'`, `resource_id = ''`, `metadata = { count: items.length }`.

**`app/api/admin/response-templates/route.ts`** (GET + POST):
- GET: include category and department names. Support `?active=true|false|all`, `?category_id`, `?department_id`.
- POST: check `name` uniqueness among active templates. AdminAuditLog: `action = 'TEMPLATE_CREATED'`.

**`app/api/admin/response-templates/[id]/route.ts`** (PATCH):
- AdminAuditLog: `action = 'TEMPLATE_UPDATED'` or `'TEMPLATE_DEACTIVATED'`.

**Error codes to return:**
- Duplicate `service_code`: `409 DUPLICATE_SERVICE_CODE`
- Duplicate dept name: `409 DUPLICATE_NAME`

- Duplicate substatus label within status: `409 DUPLICATE_SUBSTATUS_LABEL`
- Invalid foreign key references: `422 INVALID_REFERENCE`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Admin form submissions (JSON bodies) crossing into /api/admin/** route handlers |
| middleware→route | Middleware enforces admin role at edge; route handlers re-verify independently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of Privilege | All /api/admin/** route handlers | mitigate | `requireSession('admin')` called as first operation in every route handler (`app/api/admin/*/route.ts`); returns 403 if role !== 'admin'. Middleware also blocks at edge but handlers must not rely solely on middleware. |
| T-06-02 | Tampering | POST /api/admin/categories — service_code injection | mitigate | `CreateCategorySchema` enforces `regex(/^[a-zA-Z0-9_]+$/)` on service_code; Prisma parameterized queries prevent SQL injection. |
| T-06-03 | Information Disclosure | AdminAuditLog metadata — PII leakage | mitigate | `metadata` field in `prisma.adminAuditLog.create` must never include `password_hash`, `email`, `phone`, `name` fields. Route handlers only persist non-PII keys (e.g., `{ name: 'Roads', service_code: 'POTHOLE' }`). Enforced at write site in each mutation handler. |
| T-06-04 | Spoofing | Admin session cookie forgery | accept | Auth.js JWT signed with AUTH_SECRET; session cookie is httpOnly+SameSite=None+Secure. Residual risk: AUTH_SECRET compromise. Owner: deployment team (K8s secret management). |
| T-06-05 | Denial of Service | Bulk substatus reorder — large arrays | mitigate | `ReorderSubstatusSchema` validates array is min(1); route handler clamps to max 200 items. Transaction wraps all updates atomically. |
| T-06-06 | Elevation of Privilege | IDOR — PATCH /api/admin/categories/[id] | mitigate | Route handler verifies category exists via `prisma.category.findUnique({ where: { id } })`; returns 404 NOT_FOUND if missing. All mutations are admin-only so no cross-tenant risk. |
</threat_model>
