POST: Zod schema inline:
```typescript
const CreateUserSchema = z.object({
  username: z.string().min(1).max(50).regex(/^[a-zA-Z0-9_-]+$/),
  email: z.string().email(),
  password: z.string().min(12).max(200).regex(/[A-Z]/).regex(/[0-9]/),

  department_id: z.string().cuid().optional().nullable(),

```

- Hash password: `const password_hash = await bcrypt.hash(password, 12)` (import bcryptjs).
- Transaction: create User + AdminAuditLog `{ action: 'USER_CREATED', resource_type: 'User', resource_id: user.id, metadata: { username, role } }`.
- Return 201 with user (no `password_hash`).

**`app/api/admin/users/[id]/reset-password/route.ts`** (POST):

```typescript
});
```

- Hash password with bcrypt factor 12.
- Transaction:
  ```typescript

      data: {
        actor_id: session.user.id,
        action: 'USER_PASSWORD_RESET',
        resource_type: 'User',

        metadata: { note: 'Admin-initiated password reset' },

      },
    }),
  ]);
  ```

GET: Return API key list (NEVER return `key_hash` — explicit select):
```typescript
const keys = await prisma.apiKey.findMany({
  select: { id: true, label: true, scope: true, last_used_at: true, revoked_at: true, created_at: true },
  orderBy: { created_at: 'desc' },
});
```
Add `status: key.revoked_at ? 'revoked' : 'active'` to each item.

POST:
```typescript
const CreateApiKeySchema = z.object({
  label: z.string().min(1).max(100),
  scope: z.enum(['read', 'write']),
});
```
- Check label uniqueness among non-revoked keys: `409 DUPLICATE_LABEL` if exists.
- Generate plaintext key: `const plaintextKey = crypto.randomBytes(32).toString('hex');` (import `crypto` from `'node:crypto'`).
- Hash: `const key_hash = crypto.createHash('sha256').update(plaintextKey).digest('hex');`
- Transaction: create ApiKey + AdminAuditLog `{ action: 'API_KEY_GENERATED', resource_type: 'ApiKey', resource_id: key.id, metadata: { label, scope } }`.
- Return 201: `{ id, label, scope, plaintext_key: plaintextKey, created_at }`.
- **CRITICAL**: `plaintext_key` is ONLY returned in this 201 response. Never store it. Never return it from GET.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Admin form POSTs (user create, password reset, key generate) crossing into /api/admin/** |
| API→secret | Plaintext API key transiting from crypto.randomBytes → HTTP response → browser clipboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-07 | Information Disclosure | POST /api/admin/api-keys response — plaintext key leakage | mitigate | plaintext_key returned only in HTTP 201 response body; never stored in DB (only SHA-256 hash stored in `ApiKey.key_hash`); `GET /api/admin/api-keys` explicitly selects only `{ id, label, scope, last_used_at, revoked_at, created_at }` — key_hash excluded. |
| T-06-08 | Elevation of Privilege | POST /api/admin/users — role escalation | mitigate | `CreateUserSchema.role` enum-validates to `['staff', 'admin']` only; `requireSession('admin')` ensures only admins can assign roles. |
| T-06-09 | Information Disclosure | GET /api/admin/users — password_hash exposure | mitigate | Explicit Prisma `select` allowlist on User queries excludes `password_hash`. No spread operators used. |
| T-06-10 | Tampering | POST /api/admin/users/[id]/reset-password — token_version bypass | mitigate | Uses `token_version: { increment: 1 }` (Prisma atomic increment) in same transaction as password update; Auth.js middleware validates `jwt.token_version >= user.token_version` on every protected request. |
| T-06-11 | Elevation of Privilege | PATCH /api/admin/users/[id] — self-deactivation prevents admin lockout | mitigate | Route handler compares `id` param against `session.user.id`; returns `403 SELF_DEACTIVATION` if equal. Cannot be bypassed by client. |
| T-06-12 | Repudiation | Admin API key revocation — no audit trail | mitigate | Every PATCH on /api/admin/api-keys/[id] creates AdminAuditLog entry with `action: 'API_KEY_REVOKED'` in the same transaction as the revocation update. |
| T-06-13 | Information Disclosure | AdminAuditLog metadata — password hash or key hash stored | mitigate | Route handlers for password reset and API key operations explicitly set `metadata: { note: '...' }` without including sensitive fields. Code review enforced at write site. |
</threat_model>
